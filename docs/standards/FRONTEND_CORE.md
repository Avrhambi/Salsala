# 🎨 Frontend Design Core Standards

## 1. Visual Hierarchy (CRAP Principles)
* **Contrast:** Use size, weight, and color to distinguish elements. If two items are different, make them significantly different.
* **Repetition:** Maintain consistent colors, shapes, and font styles to create a predictable user experience.
* **Alignment:** Every element must have a visual connection with another element. Avoid "floating" or unaligned components.
* **Proximity:** Group related items together. Spacing should clearly indicate which label belongs to which input or image.

## 2. The 60-30-10 Color Rule
* **60% Primary:** Base/Background color.
* **30% Secondary:** Supportive elements, cards, or secondary navigation.
* **10% Accent:** Call-to-Action (CTA) buttons, notifications, and key highlights.

## 3. Interaction & Feedback
* **Affordance:** Every interactive element must have clear visual feedback for `hover`, `active`, and `disabled` states.
* **Cognitive Load:** Remove unnecessary decorations. If an item doesn't serve a clear purpose (UX), delete it.
* **Three-Click Rule:** Users should be able to find any major piece of information in no more than three actions.

## 4. Typography & Spacing
* **Hierarchy:** Max 3 fonts. Use distinct weights for headers vs. body text.
* **Rhythm:** Use consistent spacing units (e.g., multiples of 4px or 8px) for margins and padding to create a visual "beat."
